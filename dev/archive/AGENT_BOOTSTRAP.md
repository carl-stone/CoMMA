## Architecture Overview

### Data Flow
1. `commaData()` — construct object from modkit/Dorado/Megalodon output
2. `annotateSites()` — annotate methylation sites to genomic features
3. `diffMethyl()` — differential methylation testing (3 backends: beta-binomial/methylKit, limma, quasi_f)
4. `results()` / `filterResults()` — extract and filter results
5. `enrichMethylation()` — ORA/GSEA on differentially methylated genes
6. `methylomeSummary()` — per-sample QC statistics
7. `coverageDepth()` / `varianceByDepth()` — quality diagnostics
8. `slidingWindow()` — genome-wide smoothed methylation profiles
9. `mValues()` — M-value transformation for PCA
10. `writeBED()` — export to BED format

### Plot Functions (8 total)
All return ggplot/patchwork objects except `plot_heatmap()` (ComplexHeatmap):
`plot_coverage()`, `plot_methylation_distribution()`, `plot_pca()`, `plot_genome_track()`, `plot_metagene()`, `plot_tss_profile()`, `plot_volcano()`, `plot_heatmap()`

### Key Design Decisions
1. diffMethyl loops by mod_context, not mod_type — prevents spurious pooling
2. Effect sizes always on beta scale (0-1), not M-value scale
3. Multiple testing correction is genome-wide across all mod_contexts
4. annotateSites uses list-columns (CharacterList/IntegerList/NumericList) — do NOT revert to single-match
5. enrichment supports gene_role = target/regulator/both
6. KEGG offline path uses 2 bulk API calls then caches to RDS

## Hard Rules

**Always:**
- Every exported function accepts a `commaData` as primary input
- Use `GenomicRanges::findOverlaps()` for interval overlap — never nested for-loops
- Return tidy data frames or updated commaData
- All `plot_*()` return ggplot/patchwork objects
- Genome size from `commaData@genomeInfo`, never hardcoded
- Document every exported function with roxygen2: `@param`, `@return`, `@examples`
- Write testthat tests for every exported function
- Run `devtools::test()` and `devtools::document()` after changes

**Never:**
- Hardcode genome size, chromosome names, or organism-specific values
- Use nested R for-loops over genomic positions
- Import `tidyverse` — import `dplyr`, `tidyr` individually (Bioconductor requirement)
- Write stub docs for `@param`/`@return`
- Use `purrr::map_dfr()` (superseded) — use `map(...) |> list_rbind()`
- Use `purrr::map_dbl()` (superseded) — use `vapply()`

## Known Gotchas

1. **methylKit crashes** on zero-variance or all-zero-coverage sites — must filter before testing
2. **CI pinned to R 4.5** due to S4Vectors C API breakage in R 4.6.0
3. **org.EcK12.eg.db** requires `::` syntax in examples
4. **Non-ASCII characters** (e.g., ×) cause R CMD check notes
5. **`diag()` scalar trap**: `diag(x)` creates x×x identity matrix when x is scalar. Use `diag(x, nrow=1)` or `matrix(x, 1, 1)`
6. **`S4Vectors::rename()` masks `dplyr::rename()`** — always use `dplyr::rename()` explicitly
7. **`matrixStats::count()` masks `dplyr::count()`** — always use `dplyr::count()` explicitly
8. **`purrr::map()` / `mclust::map()` collision** — use `lapply()` + `purrr::list_rbind()` when mclust might be loaded
